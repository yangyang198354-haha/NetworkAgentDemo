### US-WEBUI-001: 在 Web 界面查看和筛选告警列表

- **用户故事**：As a 运维人员, I want to 在 Web 界面中查看所有告警记录的列表，并能够按告警类型、严重级别和时间范围进行筛选和分页浏览, so that 我可以快速定位需要关注的告警，不再需要通过 curl 或 Swagger 调用 API 才能查看告警数据。
- **关联需求**：REQ-WEBUI-FUNC-001, REQ-WEBUI-FUNC-002
- **优先级**：P0 (Must Have)
- **故事点**：[INFERRED — 待开发团队评估]

**验收标准：**

- **AC-W001-01** — 默认告警列表加载
  - Given 系统中存在来自 Webhook 和巡检触发的共 50 条告警记录（已持久化到 SQLite）
  - When 运维人员登录后打开"告警管理"页面
  - Then 页面应当以表格形式展示告警列表，每页默认 20 条，按发生时间倒序排列，每条记录显示：告警 ID、类型（颜色标签区分）、严重级别、设备名称、来源（WEBHOOK/INSPECTION/MOCK）、发生时间、当前状态（PROCESSING/CLOSED/FAILED/REJECTED）

- **AC-W001-02** — 多条件组合筛选
  - Given 告警列表中包含 MAC_FLAPPING、PORT_DOWN、CPU_HIGH 三种类型的告警
  - When 运维人员设置筛选条件：alert_type="PORT_DOWN"、severity="MAJOR"、时间范围为"最近 24 小时"
  - Then 列表应当仅显示同时满足三个条件的告警记录，筛选条件在界面上清晰展示，支持一键清除筛选

- **AC-W001-03** — 分页导航
  - Given 筛选后共有 45 条符合条件的告警，每页 20 条
  - When 运维人员点击"下一页"
  - Then 列表应当显示第 21-40 条记录，页面底部显示"第 2/3 页，共 45 条"，支持输入页码跳转

- **AC-W001-04** — 点击查看告警详情
  - Given 告警列表已展示
  - When 运维人员点击某条 PORT_DOWN 告警的"详情"按钮
  - Then 页面应当导航至告警详情页，展示该告警的完整处理时间线（从告警接收到最终闭环的所有节点执行记录）、诊断结果（diag_result）、根因分析（root_cause）、修复方案（fix_plan）和最终报告（final_report）

### US-WEBUI-002: 通过 Web 表单手动模拟发送告警

- **用户故事**：As a 运维人员, I want to 在 Web 界面通过下拉选择和表单填写的方式模拟发送告警（选择告警类型、目标设备、可选参数），点击按钮即可触发一次告警处理流程, so that 我不需要使用 curl 命令或 Swagger UI 手动构造 POST /alerts/simulate 请求，降低操作门槛和出错概率。
- **关联需求**：REQ-WEBUI-FUNC-003
- **优先级**：P0 (Must Have)
- **故事点**：[INFERRED — 待开发团队评估]

**验收标准：**

- **AC-W002-01** — 模拟告警表单交互
  - Given 系统中已纳管 Core-SW-01 和 Access-SW-02 两台设备
  - When 运维人员打开"模拟告警"页面，从告警类型下拉框选择="PORT_DOWN"，设备下拉框自动展示已纳管设备列表，选择="Core-SW-01"，接口名称输入="Gi0/1"
  - Then 点击"发送告警"按钮后，系统应当显示成功提示"模拟告警已发送，alert_id=xxx"，并自动跳转至告警列表页面可看到新告警记录（status=PROCESSING）

- **AC-W002-02** — 表单参数动态联动
  - Given 运维人员在模拟告警表单中选择了 alert_type="MAC_FLAPPING"
  - When 选择完成后
  - Then 表单应当动态显示 MAC 地址输入框（MAC_FLAPPING 特有参数），而 CPU 利用率输入框不显示；切换为 CPU_HIGH 时则显示 CPU 利用率输入框

- **AC-W002-03** — 必填字段校验
  - Given 运维人员未选择设备名称
  - When 点击"发送告警"按钮
  - Then 前端应当阻止提交，设备选择框高亮红色边框，提示"请选择目标设备"，请求不会发送到后端

- **AC-W002-04** — 发送后即时反馈
  - Given 运维人员正确填写了所有必填字段并点击"发送告警"
  - When 后端成功接收并触发工作流
  - Then 页面应当在 1 秒内显示成功 Toast 通知"模拟告警已发送"，并在告警列表中可立即看到该条新告警；若后端处理失败（如设备不存在），则显示错误通知并说明原因

### US-WEBUI-003: 查看告警工作流实时状态与节点拓扑

- **用户故事**：As a 运维人员, I want to 在告警详情页以图形化方式查看 LangGraph 14 节点的工作流拓扑，实时看到当前处理流程处于哪个节点、每个节点的执行状态（已完成/进行中/等待中），并能够点击节点查看其输入输出的 State 快照, so that 我可以直观地理解自动化处理的全流程，快速判断当前进度和排查异常停滞的环节。
- **关联需求**：REQ-WEBUI-FUNC-004, REQ-WEBUI-FUNC-005, REQ-WEBUI-FUNC-006
- **优先级**：P1 (Should Have)
- **故事点**：[INFERRED — 待开发团队评估]

**验收标准：**

- **AC-W003-01** — 工作流拓扑图展示
  - Given 一条 CPU_HIGH 告警正在处理中
  - When 运维人员在告警详情页切换到"工作流视图"标签
  - Then 页面应当以有向图展示 14 个节点及其连接关系：已执行节点（告警接收、告警解析、有效性校验、获取设备信息、建立SSH连接、采集诊断信息）显示为绿色"已完成"；当前执行节点（根因分析+知识库检索）以蓝色动画高亮标识"进行中"；未执行节点显示为灰色"等待中"

- **AC-W002-02** — 条件分支边展示
  - Given 工作流拓扑图已渲染
  - When 运维人员查看"风险评估"节点到"人工审批"和"备份配置"两条分支边
  - Then 分支边上应当标注流转条件文字：指向"人工审批"的边标注"need_human_approval=true"，指向"备份配置"的边标注"need_human_approval=false"，颜色区分（红色=审批路径、绿色=跳过审批路径）

- **AC-W002-03** — 节点 State 快照 Drill-down
  - Given 工作流已执行到"采集诊断信息"节点且已完成
  - When 运维人员点击拓扑图中的"采集诊断信息"节点
  - Then 右侧面板应当以 JSON 格式展示该节点的输入 State（含 alert_id、device_info）和输出 State（含新增的 diag_result），字段变更部分以黄色高亮（diff 视图），方便对比节点前后的状态变化

- **AC-W002-04** — 状态自动刷新
  - Given 告警详情页的工作流视图已打开，当前流程处于"人工审批"挂起状态
  - When 运维人员在另一个浏览器标签页中提交了审批决定（APPROVED），工作流恢复并从"人工审批"流转到"备份配置"
  - Then 告警详情页应当在 3-5 秒内自动刷新节点状态，"人工审批"节点变为绿色已完成，"备份配置"节点变为蓝色进行中

### US-WEBUI-004: 在 Web 界面完成人工审批决策

- **用户故事**：As a 运维人员, I want to 在 Web 界面查看待审批列表，点击进入审批详情页查看修复方案的完整内容（根因分析、操作步骤、影响范围、风险等级），然后通过点击"批准"或"拒绝"按钮并填写备注来完成审批决策, so that 我不需要 curl 调用审批 API，能够在一个界面中完成审批信息的查看和决策，并且审批操作有二次确认防止误触。
- **关联需求**：REQ-WEBUI-FUNC-007, REQ-WEBUI-FUNC-008
- **优先级**：P0 (Must Have)
- **故事点**：[INFERRED — 待开发团队评估]

**验收标准：**

- **AC-W004-01** — 待审批列表导航提示
  - Given 系统中有 2 条告警处于"人工审批"挂起状态
  - When 运维人员登录后查看任意页面
  - Then 侧边栏"审批管理"菜单项右上角应当显示红色徽标数字"2"，点击进入后可看到 2 条待审批项列表，每条显示：告警 ID、告警类型、设备名称、风险等级标签、挂起时长

- **AC-W004-02** — 审批详情页信息完整性
  - Given 一条 PORT_DOWN 告警的修复方案需要审批（建议在 Gi0/1 执行 `no shutdown`）
  - When 运维人员点击该待审批项进入详情页
  - Then 页面应当完整展示：告警原始描述、诊断结果（show interface 输出）、LLM 根因分析结论、修复方案详细步骤（命令列表 + 参数填充说明）、风险评估结果（风险等级 + 标记为高风险的原因）、影响范围评估

- **AC-W004-03** — 批准操作含二次确认
  - Given 运维人员查看修复方案后认为合理
  - When 运维人员点击"批准"按钮
  - Then 系统应当弹出二次确认对话框："确认批准此修复方案？该操作将立即在设备 Core-SW-01 上执行。如需备注请输入。"，运维人员确认后系统提交 approval_status=APPROVED，工作流恢复执行，页面显示"审批已提交"成功通知并返回待审批列表（该条目已移除）

- **AC-W004-04** — 拒绝操作含原因必填
  - Given 运维人员认为修复方案不可接受
  - When 运维人员点击"拒绝"按钮
  - Then 系统应当弹出对话框要求填写拒绝原因（必填，如"关键业务端口，需现场确认"），原因未填写时"确认拒绝"按钮置灰不可点击；填写后确认，系统提交 approval_status=REJECTED，告警状态更新为 REJECTED

- **AC-W004-05** — 审批操作后状态同步
  - Given 运维人员刚提交了一个审批决定
  - When 返回待审批列表页面
  - Then 该条审批项应当立即从待审批列表中消失（或标记为"已处理"），侧边栏徽标数字自动更新（减 1），审批历史页面同步新增该条记录

- **关联需求**：REQ-WEBUI-FUNC-010, REQ-WEBUI-FUNC-012

- **AC-W006-04** — 查看设备诊断结果

  - When 运维人员在设备管理页面点击 Core-SW-01 的"诊断记录"
  - Then 系统应当以代码块格式展示最近一次诊断的原始输出内容和执行时间，并标注关联的告警 ID（可点击跳转至告警详情）

- **关联需求**：REQ-WEBUI-FUNC-011

- **AC-W016-05** — Dashboard 自动刷新

  - When 新的告警被触发、审批被提交或其他状态变化发生
  - Then 统计数字和健康状态应当在下一个轮询周期（3-5 秒）内自动更新，无需手动刷新页面

### US-WEBUI-017: 通过 Web 界面登录和登出系统

- **用户故事**：As a 运维人员, I want to 在浏览器中打开 Web 界面时首先看到登录页面，输入 admin 账号和密码进行身份认证，登录成功后所有页面受 JWT Token 保护，Token 过期后自动跳转回登录页面，并能够安全登出, so that 系统操作受到身份认证保护，未授权用户无法访问任何功能页面。
- **关联需求**：REQ-WEBUI-FUNC-025, REQ-WEBUI-FUNC-026, REQ-WEBUI-NFUNC-003, REQ-WEBUI-NFUNC-004
- **优先级**：P0 (Must Have)
- **故事点**：[INFERRED — 待开发团队评估]

**验收标准：**

- **AC-W017-01** — 登录页面访问
  - Given 运维人员未登录（无有效 Token）
  - When 在浏览器地址栏输入 Web 界面的任意 URL（如 /dashboard、/alerts、/devices）
  - Then 前端路由守卫应当拦截访问，重定向到登录页面 `/login`，登录页面包含用户名输入框、密码输入框和"登录"按钮

- **AC-W017-02** — 登录成功跳转
  - Given 运维人员输入正确的 admin 账号和密码
  - When 点击"登录"
  - Then 后端验证成功返回 JWT Token，前端存储 Token，页面跳转至 Dashboard 首页，右上角显示当前用户="admin"和"退出登录"按钮

- **AC-W017-03** — 登录失败提示
  - Given 运维人员输入错误的密码
  - When 点击"登录"
  - Then 后端返回 401，登录页面显示"用户名或密码错误"的红色提示（不区分是用户名错还是密码错），密码输入框清空，用户名输入框保留已输入内容

- **AC-W017-04** — Token 过期自动跳转登录页
  - Given 运维人员的 JWT Token 已过期（签发超过 24 小时）
  - When 运维人员点击任意功能菜单或页面发起 API 请求
  - Then 后端返回 401，前端拦截器检测到后清除本地 Token，自动跳转至登录页面，并显示 Toast 提示："登录已过期，请重新登录"

- **AC-W017-05** — 安全登出
  - Given 运维人员已登录
  - When 点击右上角"退出登录"按钮
  - Then 前端清除 JWT Token，跳转至登录页面，此时通过浏览器"后退"按钮无法回到已登录页面（路由守卫拦截）
