参与流程的全部 16 个软件模块

MOD-001 WebhookReceiver

src/trigger/webhook_receiver.py

FastAPI HTTP 端点 POST /webhook/alert，接收 Mock Zabbix 告警推送，Pydantic 校验 AlertPayload 后交给 AlertNormalizer。

MOD-002 InspectionScheduler

src/trigger/inspection_scheduler.py

基于 APScheduler 的定时巡检，每 5 分钟调用 SwitchDiagTool 诊断所有纳管设备，检测到异常自动生成 RawInspectionEvent。

MOD-003 StateGraphEngine

src/orchestration/state_graph_engine.py

LangGraph 状态机引擎：14 节点 + 4 条件边 + 1 中断点。管理 run/resume/checkpoint/state query。

MOD-004 AlertNormalizer

src/orchestration/alert_normalizer.py

告警归一化器：类型映射、时效检查(TTL=15min)、去重(alert_type::device_name, 内存缓存100条)。

MOD-005 NodeHandlers

src/orchestration/node_handlers.py

14 个 LangGraph 节点处理函数，协调调用 MOD-006~015 完成所有业务逻辑。依赖注入设计，全部可单测。

MOD-006 LLMService

src/llm/llm_service.py

DeepSeek API 封装：根因分析(自由推理)、模板参数填充(纯JSON约束)、报告生成。temperature=0.1，重试3次。

MOD-007 TemplateEngine

src/llm/template_engine.py

Jinja2 沙箱模板引擎，100% 确定性，不接受任何 LLM 代码。YAML 模板 → CLI 命令列表。

MOD-008 RAGService

src/llm/rag_service.py

Chroma 向量数据库：文档嵌入、语义检索、元数据过滤(alert_type)。不可用时降级为关键词匹配 fallback。

MOD-009 OutputValidator

src/llm/output_validator.py

LLM 输出安全校验：JSON Schema 校验 + CLI 命令黑名单正则扫描。任何注入尝试触发 SECURITY_ALERT 审计。

MOD-010 SwitchConfigTool

src/tools/switch_config_tool.py

配置下发工具（策略模式：Mock + TP-Link 预留）。LangChain BaseTool 封装，逐条命令执行。

MOD-011 SwitchDiagTool

src/tools/switch_diag_tool.py

诊断命令工具（策略模式），Mock 实现为 3 种告警类型提供逼真模拟数据（MAC表/接口状态/CPU进程）。

MOD-012 BackupTool

src/tools/backup_tool.py

配置备份与回滚工具（策略模式），backup 返回模拟 running-config + UUID，rollback 模拟恢复。

MOD-013 KnowledgeBaseTool

src/tools/knowledge_base_tool.py

LangChain BaseTool 封装 MOD-008 RAGService，提供标准 Tool 接口的知识库检索。

MOD-014 RiskAssessor

src/security/risk_assessor.py

纯正则匹配风险评估：7种高风险模式(shutdown/VLAN删除/重启/路由变更等)。HIGH/CRITICAL → 强制人工审批。

MOD-015 AuditLogger

src/security/audit_logger.py

全链路审计（单例）：节点执行日志 + 不可篡改审计文件追加 + 审批挂起管理。loguru 结构化输出。

MOD-016 ConfigManager

src/security/config_manager.py

全局配置管理（单例）：YAML 加载 + 点号嵌套 key 读写 + 环境变量覆盖设备密码。线程安全。

核心数据结构 (Pydantic BaseModel)

模型文件用途关键字段
AlertPayload models/alert.py Webhook 输入 (IFC-001) alert_name, alert_type, alert_severity, alert_host, alert_ip, alert_time, alert_description, alert_interface, alert_mac, alert_cpu, event_id
Alert models/alert.py 标准化告警对象 alert_id(UUID), alert_type(enum), alert_severity, alert_content, alert_timestamp, device_info, source
DeviceInfo models/alert.py 设备信息 device_name, device_ip, device_model, interface_name, mac_address, cpu_percent
NetworkAgentState models/state.py LangGraph 状态字典 (TypedDict) alert_id, alert_type, alert_content, device_info, diag_result, root_cause, fix_plan, need_human_approval, approval_status, exec_log, verify_result, final_report, status
FixPlan models/fix_plan.py 修复方案 template_id, params{}, commands[], risk_hints[], description
ExecRecord models/fix_plan.py 单条命令执行记录 command, success, output, error, execution_time_ms, was_idempotent_skip
VerifyResult models/fix_plan.py 验证结果 verify_passed, before_state, after_state, comparison_notes
RiskAssessment models/fix_plan.py 风险评估结果 risk_level, need_human_approval, risk_reasons[], matched_high_risk_patterns[]
PendingApproval models/state.py 挂起审批项 (API 返回) checkpoint_id, alert_id, alert_type, fix_plan_summary, risk_level, risk_reasons[]
ApprovalDecision models/state.py 审批决定 (API 输入) checkpoint_id, decision("APPROVED"/"REJECTED"), operator, comment

数据存储方式（无传统关系型数据库）

存储方式用途位置/实现生命周期
LangGraph MemorySaver工作流状态检查点（State 快照），支持 Interrupt 恢复内存 (state_graph_engine.py:44)进程存活期
Chroma 向量数据库知识库文档向量索引，语义检索./data/chroma_db/（持久化）持久化
文件系统审计日志不可篡改的审计追踪（配置变更/审批/回滚/安全告警）./logs/audit.log（永久追加）持久化
文件系统操作日志每个节点的 START/END 执行记录./logs/operations_{date}.log（按日轮转，保留30天）30天
内存去重缓存AlertNormalizer 告警去重AlertNormalizer._cache（dict, TTL=15min, 最大100条）进程存活期
内存审批字典等待人工审批的记录AuditLogger._pending_approvals（dict）进程存活期
YAML 配置文件全局配置参数（巡检间隔/超时/TTL等）./config/config.yaml + 环境变量覆盖持久化
💡 关键设计决策：LLM 只填参数，不生成命令
整个流程中，LLM（DeepSeek API）仅负责两件事：根因分析（自由推理）和模板参数填充（纯 JSON 约束输出）。 实际的 CLI 命令由 TemplateEngine（Jinja2 沙箱） 确定性拼装 —— 模板是运维专家预先审核过的 YAML 文件， OutputValidator 对 LLM 输出进行 JSON Schema + CLI 黑名单双重校验，确保 LLM 的"幻觉"或"注入攻击"永远无法到达交换机执行层。
